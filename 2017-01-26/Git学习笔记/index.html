Git学习笔记

Posted by FideoJ on 2017-01-26
  • Git直接记录快照，并以校验和保存快照索引。如果文件没有修改,Git 不再重新存储该文件,而是只保留一个链接指向之前存储的文件。
  • Git 项目有三个工作区域：Git 仓库（.git directory）、工作目录（Working Directory）以及暂存区域（Staging Area）。暂存区域是一个文件。

  • .gitignore（Node项目标准.gitignore文件）

  • git commit -a -m 'added new benchmarks'

  • git rm --cached README 让文件保留在磁盘但停止跟踪
    git rm log/\*.log 删除（扩展名为.log的）文件并停止跟踪
  • git log --pretty=format:"%h - %ar : %s" --graph 一种漂亮的log格式
  • git commit --amend 修订提交信息
  • git remote show [remote-name] 远程仓库详细信息
  • git config --global alias.prettylog 'log --pretty=format:"%h - %ar : %s" --graph' Git别名
  • Git分支的实质

    首次提交的Git对象模型（无parent）

    Git 分支本质上是指向提交对象的可变指针
    git branch 得到当前分支列表
    git checkout -b [branch-name] 新建分支
    git checkout master && git merge [branch-name] 合并分支
  • 分支开发工作流

  • 跟踪分支（又称“上游分支”）是与远程分支有直接关系的本地分支

git pull 即一次git fetch加一次git merge（到跟踪分支）。

  • Git变基



变基操作的实质是丢弃一些现有的提交,然后相应地新建一些内容一样但实际上不同的提交。永远不要对在自己仓库外有副本的分支执行变基。不要用git push --force。
